## Algorithms classification for essentially contested concept morl

Based on the algorithms found in `.venv/Lib/site-packages/morl_baselines/multi_policy/`.

| Name           | Outer/Inner Loop | Main Idea | Multi/Single Policy | Action Selection | How to / if usable for ecc |
| -------------- | ---------------- | --------- | ------------------- | ---------------- | -------------------------- |
| Capql          | Inner | Extends SAC to MORL: a single actor and Q-network are conditioned on the preference weight `w`; weights are sampled from a normal-cone distribution during training so one network covers the whole simplex. Continuous actions only. | Multi (one conditioned policy network covers all weights) | Stochastic continuous action from `π(·\|s, w)`; deterministic mean at eval | Treat each interpretation of a contested concept (e.g. demographic-parity-fairness, equal-opportunity-fairness) as its own reward dimension and expand `w` accordingly. The weight-sampler then trains a single net to act consistently across all interpretations; at inference a stakeholder's interpretation profile becomes a value of `w`. Continuous actions only |
| Envelope       | Inner | Conditioned DQN: one Q-net `Q(s,a,w)` outputs a vector per action; the "envelope" target update bootstraps with the best `w'` from sampled preferences, so the agent learns the upper envelope of all linear scalarizations. | Multi (one conditioned net) | `argmax_a (w · Q(s,a,w))` (ε-greedy during training) | Same recipe: stack interpretations as extra reward components so "safety" splits into `[s¹, s², s³]` etc.; the envelope update then learns the upper envelope over linear mixes of interpretations. A new interpretation profile = a new `w` at inference, no retraining. Limited to linear aggregation across interpretations |
| GPI-PD         | Hybrid: outer Linear Support schedules training weights, inner is a conditioned Q-net using GPI | Conditioned Q-net trained on weights chosen by GPI-LS (corner weights of the CCS), with prioritized replay, Dyna updates via a probabilistic dynamics ensemble, and Generalized Policy Improvement at evaluation. | Multi (conditioned net + small set of "support" policies for GPI) | `argmax_a max_{w' ∈ W_support} (w · Q(s,a,w'))` (GPI rule) | Best deep option when the *space* of interpretations is large: corner weights produced by GPI-LS correspond to "extremal" interpretations (e.g. each interpretation taken in isolation), and GPI lets any new interpretation mix exploit the whole library. | ipro           | Outer | Iterated Pareto Referent Optimization: maintains upper/lower bounds on the Pareto front as a union of "boxes"; each iteration selects a referent and solves a scalarized subproblem (AASF) with an inner PPO solver, refining the bounds until tolerance. | Multi (one policy per subproblem; the outer loop produces a coverage set) | Delegated to the inner PPO; outer loop only chooses which referent to optimize next | Expand the reward vector to one component per interpretation; IPRO then builds a Pareto coverage set across interpretations. Because AASF is non-linear, it can produce policies in non-convex regions — useful when, e.g., two interpretations of "safety" trade off non-convexly. For ecc, pick the policy whose return vector is closest to the operationally-chosen interpretation profile |
| linear support | Outer (it's a scheduler/helper, not a full algorithm) | Builds the Convex Coverage Set by repeatedly proposing the next weight vector to train on. Two modes: OLS (Optimistic Linear Support, prioritizes weights with highest predicted scalar-value improvement) and GPI-LS (uses GPI as a proxy for value at corner weights). | Multi (drives an outer loop over weights) | N/A — it returns the next `w` to train, not actions | When interpretations are stacked as separate reward components, Linear Support enumerates the minimal set of interpretation profiles that need to be trained to cover all linear combinations. The resulting corner weights effectively answer "which combinations of interpretations are extremal enough that we need a dedicated policy for them?" |
| morld          | Outer | MORL based on Decomposition: a population of single-policy agents (MOSAC / MOSACDiscrete / EUPG) each trained on its own scalarization (weighted-sum or Tchebycheff); neighbors share buffer/parameters and optionally adapt weights (PSA). | Multi (population of `pop_size` policies, one per scalarization) | Each agent acts greedily under its scalarized critic; at inference pick the policy whose weight is closest to the target interpretation | Best fit when there is a discrete, named set of stakeholder interpretations: assign one policy per interpretation profile and let neighbors share data. With Tchebycheff scalarization, interpretations don't have to combine linearly, so e.g. a "min(fairness₁, fairness₂)" stakeholder is representable. The cost is no zero-shot generalization to unseen profiles |
| mp_mo_q_learning | Outer | Multi-Policy MO Q-Learning: trains one scalarized MOQ-Learning agent per weight; weights chosen by `random`, `ols`, or `gpi-ls`; supports Q-table transfer between weights, Dyna-Q, and a tabular GPI policy. | Multi (one Q-table per weight, optionally combined via GPI) | Greedy over scalarized Q; with `use_gpi_policy=True` it does `argmax_a max_π (w · Q_π(s,a))` over the stored policies | Closest tabular match to your original framing of "one Q-table per interpretation": train one MOQ-Learning agent per interpretation profile (or per corner weight when interpretations are stacked as reward dims), then use the GPI policy to combine them for any new profile at action time. Only practical for small discrete domains |
| pql            | Inner | Tabular Pareto Q-learning: for each `(s,a)` keep the set of non-dominated return vectors; backups propagate non-dominated combinations across successor states; an action-scoring function (hypervolume or Pareto-cardinality) drives ε-greedy selection. | Multi (the Q-set itself encodes the whole Pareto front per state) | ε-greedy over `score(s,a)` where `score` is `hypervolume(Q-set(s,a))` or `\|non_dom(Q-set(s,a))\|` | Strongest conceptual fit: every interpretation becomes a coordinate of the Pareto Q-set, and the agent never needs to commit to an aggregation across interpretations. Any non-linear, contested aggregation (e.g. "as long as no interpretation falls below threshold τ") can be evaluated against the stored Pareto set post-hoc. Cost: dimensionality of the set grows with the number of interpretations, so practical only for a handful per concept and small state spaces |
| pcn            | Inner | Pareto Conditioned Network: "upside-down" RL — a single network is trained by supervised learning to map `(state, desired_return, desired_horizon)` to actions, using a replay of Pareto non-dominated episodes filtered by crowding distance. | Multi (one network covers a continuum of desired returns) | Forward pass of the conditioned network; action `= π(s, desired_return, desired_horizon)` | After stacking interpretations as reward dims, an interpretation profile becomes a target vector in `desired_return` (e.g. "I want fairness¹ ≥ 0.9 and safety² ≥ 0.7"). The net interpolates over the Pareto archive, so even idiosyncratic combinations a stakeholder cares about are reachable if they lie inside the learned front. Good fit when interpretations don't combine linearly |
| pgmorl         | Outer (evolutionary) | Prediction-Guided MORL: keeps a population of MO-PPO agents; a hyperbolic performance-predictor learns Δreturn given current return and weight; each generation assigns `(agent, weight)` pairs to maximize predicted hypervolume improvement. | Multi (population of PPO policies → final Pareto archive) | Each agent acts via PPO; at inference pick the archived policy that best matches the interpretation | Similar to MORL/D but with evolutionary task-weight assignment and continuous control. Use when interpretations are stacked as reward dims and you want the resulting Pareto archive to densely cover the interpretation simplex. Less flexible than weight-conditioned methods for unseen profiles — you pick from the archive |

### Quick takeaways for ecc

The general pattern: **stack each interpretation of a contested concept as its own reward dimension**, then ask how the algorithm treats that expanded reward vector. Different algorithm families correspond to different assumptions about how stakeholders combine interpretations:

* **Aggregation across interpretations is linear and may vary across stakeholders** → use a weight-conditioned net (Envelope, CAPQL, GPI-PD). Any stakeholder's interpretation profile is a value of `w` at inference. GPI-PD is the strongest if interpretations are many.
* **Aggregation is non-linear or hard to elicit** → use a Pareto-set method (PQL, PCN). The set of non-dominated return vectors stores every interpretation coordinate without committing to an aggregation rule; downstream you can apply any rule — minimum, threshold, lexicographic — post-hoc.
* **There is a discrete set of named interpretation profiles** → use a population method (MORL/D, MP-MOQL, PGMORL, IPRO). One trained policy per profile. Tchebycheff (MORL/D) and AASF (IPRO) cover non-convex regions; the others assume linear scalarization.
* `linear_support` is not a stand-alone algorithm — it's the corner-weight scheduler underpinning GPI-PD and MP-MOQL, and answers "which interpretation profiles are extremal enough to deserve their own training run?"
